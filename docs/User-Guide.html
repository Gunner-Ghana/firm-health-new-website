1. Website Overview

The Firm Health Ghana Foundation website is a modern, responsive web application designed to showcase your organization's mission, programs, and impact. The website consists of two main parts:

Public Website The front-facing pages that visitors see, including information about your programs, team, events, and ways to get involved.
Admin Dashboard A secure, password-protected area where authorized users can manage all website content.

Key Features

  • Volunteer registration and management
  • Blog publishing system
  • Event management with calendar
  • Intervention programs showcase
  • Photo gallery with categories
  • Board and team member profiles
  • Responsive design for all devices

Website URL: https://firm-health-new-website.vercel.app/

3. Accessing the Admin Dashboard

The admin dashboard is a secure area where you can manage all content on your website. Only authorized users with the correct credentials can access it.

How to Access the Dashboard

  1. Open your web browser
    Use any modern browser like Chrome, Firefox, Safari, or Edge.
  2. Navigate to the admin login page
    Go to: https://firm-health-new-website.vercel.app/admin
  3. Enter your login credentials
    Type your username and password in the login form.
  4. Click the "Login" button
    You will be redirected to the dashboard if your credentials are correct.

Default Login Credentials

Username: admin

Password: admin123

Security Tip: It is recommended to change these default credentials in production. Contact your developer to update the login credentials for better security.

Admin Login Page

5. Managing Volunteers

The Volunteer Management section allows you to view, manage, and feature volunteers who have signed up through the website.

Accessing Volunteer Management

  1. Click "Volunteer Management" in the sidebar
    This opens the volunteer management page.

Volunteer List View

You'll see a grid of volunteer cards showing:

  • Profile photo (or placeholder with initials)
  • Name and email
  • Registration date
  • Skills and availability
  • Status (Active, Pending, Inactive)

Viewing Volunteer Details

  1. Find the volunteer card
    Scroll through the grid to find the volunteer you want to view.
  2. Click "View Details"
    A modal will open showing the volunteer's complete information including their biography and skills.

Featuring a Volunteer on the Website

You can choose to display certain volunteers on the public website's volunteer page.

  1. Find the volunteer card
    Locate the volunteer you want to feature.
  2. Click "Show on Website" toggle
    When enabled (green), the volunteer will appear on the public Volunteers page.

Where it appears: Featured volunteers are displayed at /management/volunteers on the public website.

Editing Volunteer Information

  1. Click the "Edit" button on the volunteer card
    This opens the edit form.
  2. Modify the information
    Update name, email, skills, biography, or upload a new photo.
  3. Click "Save Changes"
    Your changes will be saved and reflected immediately.

Changing Volunteer Status

Volunteers can have three statuses:

Status Description
Active Currently participating volunteers
Pending New signups awaiting approval
Inactive Former or paused volunteers

Deleting a Volunteer

  1. Click the "Delete" button
    A confirmation prompt will appear.
  2. Confirm the deletion
    Click "Confirm Delete" to permanently remove the volunteer.

Warning: Deleting a volunteer is permanent and cannot be undone.

6. Managing Blog Posts

The Blog Management section allows you to create, edit, and publish blog posts that appear on your website.

Accessing Blog Management

  1. Click "Blog Management" in the sidebar
    This opens the blog management page.

Creating a New Blog Post

  1. Click the "Create New Post" button
    A form will appear for creating a new blog post.
  2. Enter the blog title
    This will be the headline displayed on the blog listing and detail pages.
  3. Add a featured image
    Click the upload area to select an image from your computer. This image appears at the top of the blog post.
  4. Write the blog content
    Enter the main body of your blog post in the content area.
  5. Add an excerpt (optional)
    This short summary appears on the blog listing page.
  6. Choose publish status
    Check "Published" to make it visible on the website, or leave unchecked to save as a draft.
  7. Click "Save Post"
    Your blog post will be saved.

Where it appears: Published blog posts appear at /blogs and individual posts at /blog/[post-slug]

Editing an Existing Blog Post

  1. Find the blog post in the list
    Posts are displayed as cards with the title and featured image.
  2. Click the "Edit" button
    The edit form will open with the current content.
  3. Make your changes
    Update the title, content, image, or publish status.
  4. Click "Save Changes"
    Your updates will be saved and reflected on the website.

Publishing and Unpublishing

You can control whether a blog post is visible to the public:

  • Published: Visible on the public website
  • Draft: Only visible in the admin dashboard

Toggle the "Published" checkbox to change the status.

Deleting a Blog Post

  1. Click the "Delete" button on the blog card
    A confirmation prompt will appear.
  2. Confirm the deletion
    The post will be permanently removed.

7. Managing Events

The Event Management section allows you to create and manage events that will be displayed on your website's events page.

Accessing Event Management

  1. Click "Event Management" in the sidebar
    This opens the event management page.

Creating a New Event

  1. Click "Create New Event"
    The event creation form will appear.
  2. Enter event details:
    • Title: Name of the event
    • Date: When the event will take place
    • Time: Start time of the event
    • Location: Where the event will be held
    • Type: Category (e.g., Health Screening, Blood Donation, Community Outreach)
  3. Add an event image
    Upload a promotional image for the event.
  4. Write the event description
    Provide details about what attendees can expect.
  5. Set the publish status
    Check "Published" to make it visible on the website.
  6. Click "Save Event"
    Your event will be created.

Where it appears: Published events appear at /events on the public website.

Event Types

Events are categorized by type for easy filtering:

Type Description
Health Screening Medical checkup and screening events
Blood Donation Blood donation drives
Community Outreach Community engagement activities
Training Educational and training sessions
Fundraising Fundraising events

Editing Events

  1. Find the event card
    Events are displayed with their date, title, and image.
  2. Click "Edit"
    Make your changes to any field.
  3. Save your changes
    Click "Save Event" to update.

Upcoming vs Past Events

The website automatically categorizes events based on their date:

  • Upcoming Events: Events with dates in the future
  • Past Events: Events with dates that have passed

8. Managing Interventions

The Interventions section allows you to showcase your organization's programs and initiatives.

Accessing Interventions Management

  1. Click "Interventions" in the sidebar
    This opens the interventions management page.

Creating a New Intervention

  1. Click "Create New Intervention"
    The creation form will appear.
  2. Enter intervention details:
    • Title: Name of the intervention/program
    • Slug: URL-friendly identifier (auto-generated from title)
    • Featured Image: Main image for the program
    • Short Description: Brief overview shown in listings
    • Full Description: Detailed information about the program
  3. Add objectives and impact
    Describe the goals and outcomes of the intervention.
  4. Set display order
    Choose the order in which this intervention appears relative to others.
  5. Publish the intervention
    Check "Published" to make it visible.
  6. Click "Save"
    Your intervention will be created.

Where it appears: Published interventions appear at /interventions/[slug] on the public website.

Static vs Dynamic Interventions

The website has some pre-built intervention pages (like ICA Programmes, Blood Donation, etc.) and also supports dynamic interventions that you create through the dashboard.

Static Pages Dynamic Pages
Pre-designed with specific layouts Created through the dashboard
ICA Programmes, Blood Donation, Medical Screening, Feed The Aged Any new intervention you create
Content is fixed in code Content is fully editable

Reordering Interventions

Use the "Order" field to control the display sequence of interventions. Lower numbers appear first.

10. Managing Board & Team Members

The Board & Team section allows you to manage the profiles of your organization's board of directors and management team.

Accessing Member Management

  1. Click "Board & Team" in the sidebar
    This opens the member management page.

Understanding the Tabs

The page has two tabs:

Tab Description Displays On
Board Members Members of the Board of Directors /management/board
Team Members Management team staff /management/teams

Adding a New Member

  1. Select the appropriate tab
    Choose "Board Members" or "Team Members" depending on which category the person belongs to.
  2. Click "Add New Member"
    The member form will appear.
  3. Fill in the member details:
    • Name: Full name of the member
    • Email: Contact email address
    • Role: Their title (e.g., "Chairman", "Executive Director")
    • Position: Additional position info (e.g., "Medical Doctor")
  4. Upload a profile photo
    Click the upload area to add their photo.
  5. Add biography information:
    • Short Bio: Brief description shown on the card
    • Full Bio: Complete biography shown in the modal
  6. Set publish status
    Check "Published" to display this member on the website.
  7. Click "Save Member"
    The member will be added.

Add Member Form

Form with fields for name, email, role, position, photo upload, short bio, and full bio. A "Published" checkbox at the bottom controls visibility.

Where it appears:

  • Board Members → /management/board
  • Team Members → /management/teams

How Members Display on the Website

On the public website, members are displayed as cards showing:

  • Profile photo (or placeholder if none uploaded)
  • Name and role
  • Position
  • Email (clickable)
  • Short bio
  • "See Biography" button that opens a modal with the full bio

Editing a Member

  1. Find the member card
    Locate the member you want to edit.
  2. Click "Edit"
    The form will open with their current information.
  3. Make your changes
    Update any fields as needed.
  4. Click "Save Changes"
    Your updates will be saved.

Publishing and Unpublishing Members

Use the "Toggle Publish" button to control visibility:

  • Published: Member appears on the public website
  • Draft: Member is hidden from the public site but remains in the dashboard

Deleting a Member

  1. Click "Delete" on the member card

  2. Click "Confirm Delete"

12. Troubleshooting & FAQs

Common Issues and Solutions

Q: I can't log in to the admin dashboard

A: Make sure you're using the correct credentials (username: admin, password: admin123). Check that Caps Lock is not enabled. If you've changed the password and forgotten it, contact your developer.

Q: I added content but it's not showing on the website

A: Make sure the content is marked as "Published". Only published content appears on the public website. Check the publish status checkbox when editing the item.

Q: Images are not uploading

A: Ensure your image file is in a supported format (JPG, PNG, GIF, WebP). Try reducing the file size if it's very large. Clear your browser cache and try again.

Q: The page shows "No items found"

A: This means no content has been added yet, or all content is set to draft/unpublished. Add new content or publish existing drafts.

Q: Changes are not reflecting immediately

A: Try refreshing the page (Ctrl+F5 or Cmd+Shift+R). If using a CDN or caching, changes may take a few minutes to propagate.

Q: I accidentally deleted something

A: Unfortunately, deletions are permanent. Always double-check before confirming a deletion. Consider keeping backups of important content.

Best Practices

  • Use high-quality images: Photos should be clear and well-lit. Recommended minimum size is 800x600 pixels.
  • Write descriptive titles: Clear titles help visitors understand the content at a glance.
  • Keep content updated: Regularly update events, remove outdated content, and add new blog posts to keep the site fresh.
  • Preview before publishing: After publishing, visit the public page to verify everything looks correct.
  • Organize photos by category: Use appropriate categories for photos to help visitors filter the gallery.

Getting Help

If you encounter issues not covered in this guide:

  1. Check if the issue is covered in this documentation
  2. Try refreshing the page or clearing browser cache
  3. Contact your website developer for technical support

Technical Support Contact: For technical issues, please contact your web developer with a detailed description of the problem, including any error messages you see.
